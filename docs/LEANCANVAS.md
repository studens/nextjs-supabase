## 1. 문제 (Problem)

> 소규모 정기 모임 주최자가 카카오톡 단톡방에서 겪는 3가지 핵심 문제

**① 공지 유실**

- 이벤트 공지 메시지가 대화 속에 묻혀 참여자가 놓치는 경우가 빈번
- 중요한 날짜/장소 정보를 반복해서 올려야 하는 번거로움

**② 참여 확인 수작업**

- "참가요" / "못 가요" 메시지를 일일이 스크롤하며 집계
- 마감 직전 응답 변경 시 재집계 필요

**③ 의사결정 비효율**

- 날짜·장소·메뉴 등을 댓글로 투표할 때 집계 오류 발생
- 최종 결정 사항이 어디 있는지 찾기 어려움

**기존 대안**:

- 카카오톡 단톡방 (가장 보편적, 위 문제 그대로 존재)
- 네이버 밴드 (기능 복잡, 소규모 모임에 과함)
- 구글 폼 (매번 새로 만들어야 함, 히스토리 없음)

---

## 2. 고객 세그먼트 (Customer Segments)

**주요 타깃: 소규모 정기 모임 주최자 (Host)**

- 10~50명 규모의 정기 모임을 운영하는 사람
- 골프 동호회, 수영 모임, 헬스 그룹, 친목 모임 등
- 매월 1회 이상 정기적으로 모임을 주최하는 사람
- 스마트폰·PC 기본 사용 가능한 30~50대

**보조 타깃: 모임 멤버 (Member)**

- 주최자의 초대를 받아 참여하는 사람
- 공지 확인, 참여 응답, 투표 참여가 주요 행동

**얼리 어답터**:

- 카카오톡 단톡방 관리에 피로감을 느끼는 동호회 운영진
- 디지털 도구에 친숙한 30~40대 모임 주최자

---

## 3. 가치 제안 (Unique Value Proposition)

> **"모임 운영의 모든 것을 한 곳에서 — 공지, 참여 확인, 투표"**

- 카카오톡 단톡방에서 흩어지던 공지·참여 확인·투표를 **하나의 웹 앱**에서 처리
- 주최자는 **클릭 몇 번**으로 이벤트를 만들고, 참여 현황을 실시간으로 확인
- 멤버는 **이메일 링크 한 번**으로 바로 참여 응답 가능 (가입 필요하지만 간편)
- 모임 이력이 **누적·보관**되어 언제든 이전 이벤트 확인 가능

---

## 4. 해결책 (Solution)

**핵심 기능 3가지**

| 기능                        | 해결하는 문제                   |
| --------------------------- | ------------------------------- |
| **이벤트 공지 & 참여 관리** | 공지 유실, 참여 확인 수작업     |
| **투표 시스템**             | 의사결정 비효율                 |
| **그룹 & 멤버 관리**        | 반복적인 멤버 초대 및 권한 관리 |

**구체적 해결 방식**:

- 이벤트 생성 → 이메일 알림 자동 발송 → 참가/불참/미정 원클릭 응답
- 투표 생성 → 선택지별 실시간 집계 → 주최자가 최종 결정 확정
- 그룹별 이벤트·투표 히스토리 누적 보관

---

## 5. 경쟁우위 (Unfair Advantage)

> 쉽게 복사하거나 구매할 수 없는 차별점

- **소규모 모임 특화**: 네이버 밴드·카페 같은 대형 플랫폼과 달리 10~50명 소규모 정기 모임에 최적화된 UX
- **카카오톡 초대 친화적**: 링크 공유만으로 초대 가능한 진입 장벽 최소화 설계
- **주최자 중심 워크플로우**: 공지→참여확인→투표의 반복 흐름을 단일 앱에서 완결
- **이력 관리**: 지난 모임 기록이 자동 누적되어 운영 연속성 보장

---

## 6. 수익 흐름 (Revenue Stream)

> MVP 단계에서는 무료 운영, 이후 단계적 수익화

**Phase 1 (MVP): 무료**

- 사용자 확보 및 제품-시장 적합성(PMF) 검증에 집중
- 광고 없는 순수 무료 서비스

**Phase 2 (수익화 검토)**

- **Freemium 모델**:
  - 무료: 그룹 1개, 멤버 최대 20명, 이벤트 월 5개
  - 유료(Pro): 그룹 무제한, 멤버 무제한, 이벤트 무제한, 이메일 알림 커스텀
- **가격 예시**: 월 5,000~9,900원 / 그룹 (주최자 결제)

**Phase 3 (확장)**

- 팀/기업 플랜 (동호회 연합, 사내 소모임 등)
- 이벤트 장소 제휴 광고 (골프장, 수영장 등)

## 7. 비용 구조 (Cost Structure)

**인프라 비용 (월 기준)**
| 항목 | 비용 (예상) |
|------|------------|
| Supabase (DB + Auth + Storage) | 무료 ~ $25/월 |
| Vercel (호스팅) | 무료 ~ $20/월 |
| 이메일 발송 (Resend) | 무료 ~ $20/월 (3,000건/월 무료) |
| **합계** | **$0 ~ $65/월** |

**개발 비용**

- 1인 개발 (솔로 개발자): 인건비 최소화
- 오픈소스 UI 라이브러리 활용 (shadcn/ui, Tailwind CSS)

**MVP 총 초기 비용**: 도메인 비용 ($10~15/년) 외 거의 무료로 운영 가능

## 8. 주요 지표 (Key Metrics)

**획득 (Acquisition)**

- 신규 가입 그룹 수 (목표: 론칭 후 1개월 내 10개 이상)
- 회원가입 전환율 (초대 이메일 수신 → 가입 완료)

**활성화 (Activation)**

- 그룹 생성 후 첫 이벤트 생성까지의 시간
- 이벤트당 평균 RSVP 응답률 (목표: 70% 이상)

**유지 (Retention)**

- 주 1회 이상 사용하는 그룹 비율 (목표: 50% 이상)
- 주최자 월간 재사용률 (목표: 60% 이상)

**추천 (Referral)**

- 그룹당 평균 초대 발송 수
- 초대 수락률

**수익 (Revenue)**

- MVP 단계: 해당 없음 (무료)
- 이후: 유료 전환율, ARPU (사용자당 평균 매출)

---

## 9. 채널 (Channels)

**초기 확보 채널 (무료)**

- **지인 네트워크**: 직접 아는 동호회 주최자에게 먼저 소개
- **카카오오픈채팅 / 네이버 카페**: 골프·수영·헬스 동호회 커뮤니티 홍보
- **블로그/SNS**: 개발 과정 공유 (개발 블로그, X/트위터)

**바이럴 채널 (제품 내 내재)**

- **초대 이메일**: 주최자가 멤버를 초대할 때 자연스럽게 서비스 노출
- **이벤트 공유 링크**: 멤버가 비가입자에게 이벤트 링크 공유 시 신규 유입

**후기 채널 (중장기)**

- 앱 스토어 리뷰 (모바일 앱 출시 이후)
- Product Hunt 등 스타트업 커뮤니티
