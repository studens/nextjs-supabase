# 모임 이벤트 관리 웹 MVP PRD

> 작성일: 2026-04-14

---

## 핵심 정보

**목적**: 소규모 정기 모임(골프·수영·친목 등)에서 카카오톡 단톡방으로 처리하던 공지·참여 확인·투표를 한 곳에서 효율적으로 관리할 수 있게 한다.

**사용자**: 10~50명 규모의 소규모 정기 모임을 운영하는 주최자(Host)와 초대받아 참여하는 멤버(Member).

---

## 사용자 여정

### Flow A — 이벤트 공지 (주최자)

```
1. 로그인 페이지
   ↓ 이메일/비밀번호 또는 구글 OAuth 로그인

2. 내 그룹 목록 페이지 (홈)
   ↓ 그룹 카드 클릭

3. 그룹 홈 페이지 (이벤트 목록)
   ↓ "이벤트 생성" 버튼 클릭

4. 이벤트 생성/수정 페이지
   ↓ 제목·날짜·장소·내용 입력 후 저장
   → 멤버에게 이메일 알림 자동 발송

5. 이벤트 상세 페이지 (공지 + 참여 응답)
   ↓ "참여 현황 보기" 클릭

6. 참여 현황 상세 페이지
   ↓ 상태별 인원·명단 확인 → 완료
```

### Flow B — 참여 응답 (멤버)

```
1. 이메일 알림 수신
   ↓ 이메일 내 링크 클릭

2. 로그인 페이지
   ↓ 로그인 완료

3. 이벤트 상세 페이지 (공지 + 참여 응답)
   ↓ 참가 / 불참 / 미정 버튼 선택

4. 이벤트 상세 페이지
   → 내 응답 상태 반영 확인 → 완료
```

### Flow C — 투표 (주최자·멤버)

```
1. 그룹 홈 페이지 (이벤트 목록)
   ↓ 투표 탭 클릭

2. 투표 목록 페이지
   [주최자] ↓ "투표 생성" 버튼 클릭
       → 투표 생성/수정 페이지 → 선택지 입력 후 저장 → 투표 상세 페이지
   [멤버]  ↓ 투표 항목 클릭
       → 투표 상세 페이지

3. 투표 상세 페이지 (투표 + 결과)
   ↓ 선택지 선택 후 투표 제출

4. 투표 상세 페이지
   → 실시간 결과(비율 바 차트) 확인

   [주최자] ↓ "최종 결정" 또는 "수동 마감" 클릭 → 완료
```

### Flow D — 멤버 초대

```
1. 멤버 관리 페이지 (주최자)
   ↓ 이메일 입력 후 초대 발송

2. 수신자 이메일 수신
   ↓ 초대 링크 클릭

3. 로그인/회원가입 페이지
   ↓ 신규: 회원가입 / 기존: 로그인

4. 내 그룹 목록 페이지 (홈)
   → 해당 그룹 자동 합류 → 그룹 홈 페이지로 이동
```

### Flow E — 구글 OAuth 로그인

```
1. 로그인 페이지
   ↓ "구글로 로그인" 버튼 클릭

2. 구글 계정 선택 화면 (외부)
   ↓ 계정 선택

3. OAuth 콜백 처리
   ↓ 신규 사용자 자동 계정 생성

4. 내 그룹 목록 페이지 (홈)
   → 로그인 완료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                 | 설명                                                                     | MVP 필수 이유                               | 관련 페이지                               |
| -------- | ---------------------- | ------------------------------------------------------------------------ | ------------------------------------------- | ----------------------------------------- |
| **F001** | 그룹 생성              | 이름·설명·카테고리(골프/수영/헬스/친목/기타)를 입력해 모임 그룹 생성     | 서비스의 최상위 단위, 없으면 모든 기능 불가 | 그룹 생성 페이지                          |
| **F002** | 멤버 초대              | 이메일 입력 → 초대 이메일 발송 → 가입 시 그룹 자동 합류                  | 모임 멤버를 모으는 핵심 진입 경로           | 멤버 관리 페이지                          |
| **F003** | 이벤트 생성·수정·취소  | 제목·날짜·장소·내용(마크다운)·상태·RSVP 마감일 입력 및 수정              | 서비스의 핵심 콘텐츠, 주최자의 핵심 작업    | 이벤트 생성/수정 페이지                   |
| **F004** | 이벤트 공지 열람       | 그룹 멤버가 예정·진행 중·완료 이벤트 목록과 상세 내용 확인               | 멤버의 1차 사용 목적                        | 그룹 홈 페이지, 이벤트 상세 페이지        |
| **F005** | 참여 응답(RSVP)        | 이벤트에 참가·불참·미정 응답 및 변경                                     | 공지와 함께 서비스의 핵심 가치              | 이벤트 상세 페이지                        |
| **F006** | 참여 현황 확인         | 상태별(참가·불참·미정·미응답) 인원 수와 명단 조회                        | 주최자의 모임 운영 의사결정에 직결          | 이벤트 상세 페이지, 참여 현황 상세 페이지 |
| **F007** | 투표 생성·수정·마감    | 제목·선택지(2~6개)·단일/복수 선택·마감일 설정, 수동 마감, 최종 결정 표시 | 날짜·장소 결정 시 핵심 도구                 | 투표 생성/수정 페이지                     |
| **F008** | 투표 참여 및 결과 확인 | 선택지 선택 후 제출, 선택지별 투표수·비율 바 차트 실시간 확인            | 투표 기능의 멤버 측 핵심 경험               | 투표 상세 페이지                          |

### 2. MVP 필수 지원 기능

| ID       | 기능명         | 설명                                                         | MVP 필수 이유                                               | 관련 페이지                                                             |
| -------- | -------------- | ------------------------------------------------------------ | ----------------------------------------------------------- | ----------------------------------------------------------------------- |
| **F010** | 기본 인증      | 이메일/비밀번호 회원가입·로그인·로그아웃 + 구글 OAuth 로그인 | 서비스 이용을 위한 최소 인증, 구글 OAuth는 초대 흐름에 필수 | 로그인 페이지, 회원가입 페이지                                          |
| **F011** | 그룹 설정      | 그룹 이름·설명·카테고리 수정 (주최자 전용)                   | 그룹 기본 정보 유지 관리                                    | 그룹 설정 페이지                                                        |
| **F012** | 멤버 역할 관리 | 그룹 내 Host/Member 역할 구분, 권한 제어                     | 주최자 전용 기능(이벤트 생성·멤버 초대 등) 보호             | 멤버 관리 페이지                                                        |
| **F013** | 이메일 알림    | 이벤트 생성·수정, 투표 생성 시 그룹 멤버 이메일 발송         | 별도 채널(카톡) 없이 멤버에게 알림 전달                     | (백엔드 처리, 이벤트 생성/수정 페이지·투표 생성/수정 페이지에서 트리거) |

### 3. MVP 이후 기능 (제외)

- 카풀 매칭
- 비용 정산

- 모바일 앱

## 메뉴 구조

```
모임 이벤트 관리 앱 내비게이션

공개 메뉴 (비로그인)
├── 로그인 페이지
│   └── 기능: F010 (이메일/비밀번호 로그인, 구글 OAuth 로그인)
└── 회원가입 페이지
    └── 기능: F010 (이메일/비밀번호 회원가입, 구글 OAuth 신규 가입)

├── 내 그룹 목록 페이지 (홈)

│   └── 기능: F001 (그룹 생성 폼)

그룹 멤버 메뉴 (그룹 입장 후)
├── 그룹 홈 페이지 (이벤트 목록 탭)
│   └── 기능: F004 (이벤트 목록 조회)

│   └── 기능: F008 (투표 목록 조회 및 투표 상세 진입)
├── 이벤트 상세 페이지
│   └── 기능: F004, F005, F006 (이벤트 공지 열람, 참여 응답, 참여 현황 요약)
└── 투표 상세 페이지
    └── 기능: F008 (투표 참여 및 결과 확인)

주최자(Host) 전용 메뉴 (그룹 입장 후)
├── 이벤트 생성/수정 페이지
│   └── 기능: F003, F013 (이벤트 생성·수정·취소, 이메일 알림 트리거)
├── 참여 현황 상세 페이지
│   └── 기능: F006 (상태별 상세 명단 조회)
├── 투표 생성/수정 페이지
│   └── 기능: F007, F013 (투표 생성·수정·마감, 이메일 알림 트리거)
├── 멤버 관리 페이지
│   └── 기능: F002, F012 (멤버 초대, 역할 관리)
└── 그룹 설정 페이지
    └── 기능: F011 (그룹 이름·설명·카테고리 수정)
```

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F010` | **접근:** 비로그인 공개

| 항목            | 내용                                                                                                                                                                                                  |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 서비스 진입을 위한 인증 페이지 (이메일 로그인 + 구글 OAuth 로그인)                                                                                                                                    |
| **진입 경로**   | 앱 최초 접근 시, 비로그인 상태로 보호된 페이지 접근 시 자동 리디렉션                                                                                                                                  |
| **사용자 행동** | 이메일/비밀번호 입력 후 로그인 버튼 클릭, 또는 구글 버튼 클릭 후 OAuth 인증                                                                                                                           |
| **주요 기능**   | • 이메일/비밀번호 입력 폼 및 유효성 검사<br>• 구글 OAuth 로그인 버튼 (Supabase Auth)<br>• 로그인 실패 시 오류 메시지 표시<br>• 회원가입 페이지 이동 링크<br>• 비밀번호 찾기 링크<br>• **로그인** 버튼 |
| **다음 이동**   | 성공 → 내 그룹 목록 페이지 (홈), 실패 → 오류 메시지 표시 후 유지                                                                                                                                      |

### 회원가입 페이지

> **구현 기능:** `F010` | **접근:** 비로그인 공개

| 항목            | 내용                                                                                                                                                                                         |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 신규 사용자 계정 생성 페이지 (이메일 가입 + 구글 OAuth 자동 가입)                                                                                                                            |
| **진입 경로**   | 로그인 페이지에서 "회원가입" 링크 클릭, 또는 초대 이메일 링크 클릭 후 신규 사용자로 판단 시 리디렉션                                                                                         |
| **사용자 행동** | 이름·이메일·비밀번호 입력 후 가입, 또는 구글 버튼으로 자동 가입                                                                                                                              |
| **주요 기능**   | • 이름·이메일·비밀번호 입력 폼 및 유효성 검사<br>• 구글 OAuth 가입 버튼<br>• 이메일 인증 안내 메시지<br>• 초대 토큰이 있는 경우 가입 완료 후 해당 그룹 자동 합류 처리<br>• **회원가입** 버튼 |
| **다음 이동**   | 성공 → 내 그룹 목록 페이지 (홈) 또는 초대받은 그룹 홈 페이지, 실패 → 오류 메시지 표시                                                                                                        |

### 내 그룹 목록 페이지 (홈)

> **구현 기능:** `F001` | **접근:** 로그인 필수

| 항목            | 내용                                                                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| **역할**        | 로그인 후 최초 도착하는 홈 화면으로, 내가 속한 모든 그룹을 조회하고 새 그룹을 만드는 기점                                                        |
| **진입 경로**   | 로그인 성공 후 자동 리디렉션, 헤더 로고/홈 클릭                                                                                                  |
| **사용자 행동** | 그룹 카드 목록 확인, 특정 그룹 클릭해서 진입, 새 그룹 생성 버튼 클릭                                                                             |
| **주요 기능**   | • 내가 속한 그룹 카드 목록 표시 (그룹명·카테고리·멤버 수·최근 이벤트 날짜)<br>• 그룹 없을 시 빈 상태(empty state) 안내<br>• **그룹 만들기** 버튼 |
| **다음 이동**   | 그룹 카드 클릭 → 그룹 홈 페이지, 그룹 만들기 클릭 → 그룹 생성 페이지                                                                             |

### 그룹 생성 페이지

> **구현 기능:** `F001` | **접근:** 로그인 필수

| 항목            | 내용                                                                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 새 모임 그룹을 생성하는 폼 페이지                                                                                                                               |
| **진입 경로**   | 내 그룹 목록 페이지에서 "그룹 만들기" 버튼 클릭                                                                                                                 |

> **구현 기능:** `F003`, `F004`, `F007`, `F008` | **접근:** 그룹 멤버 (로그인 필수)

| 항목            | 내용                                                                                                                                                                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 그룹의 메인 화면으로, 이벤트 목록과 투표 목록을 탭으로 구분해 제공                                                                                                                                                                                              |

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) — React 풀스택 프레임워크, 서버 컴포넌트 및 서버 액션 활용
- **TypeScript 5.6+** — 타입 안전성 보장
- **React 19** — UI 라이브러리 (최신 동시성 기능)

### 스타일링 & UI

- **TailwindCSS v4** (설정 파일 없는 새로운 CSS 엔진) — 유틸리티 CSS 프레임워크
- **shadcn/ui** — 고품질 React 컴포넌트 라이브러리 (New York 스타일)
- **Lucide React** — 아이콘 라이브러리

### 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod** — 스키마 검증 라이브러리

### 백엔드 & 데이터베이스

- **Supabase** — BaaS (인증·데이터베이스·Row Level Security)
- **PostgreSQL** — 관계형 데이터베이스 (Supabase 포함)
- **Supabase Auth** — 이메일/비밀번호 + 구글 OAuth 인증
- **RLS (Row Level Security)** — 그룹 멤버만 그룹 데이터 접근 가능하도록 정책 설정

### 이메일 알림

- **Resend** (또는 Supabase Email) — 이벤트·투표 생성 시 멤버 이메일 알림 발송

### 배포 & 호스팅

- **Vercel** — Next.js 15 최적화 배포 플랫폼

### 패키지 관리

- **npm** — 의존성 관리
